# Committed tearsheets, byte-identical regen

This case study ships the following **committed** files (repo root):

```
manuscripts/
├── FINDINGS.md                    ← auto-generated by notebooks/06_synthesis.py, committed
├── DIAGNOSTICS_CHECKLIST.md       ← auto-generated by notebooks/06_synthesis.py, committed
├── tearsheets/
│   ├── 01_load_and_preprocess.md  ← one per notebook, exported, committed
│   ├── 02_balance_and_pretrends.md
│   ├── 03_main_effects.md
│   └── …                          (through 14_dohmh_secondary_outcome.md)
├── MANUSCRIPT.md                  ← hand-authored paper, committed
├── AUDIT.md                       ← hand-authored self-critique, committed
├── METHODOLOGY.md                 ← hand-authored, committed
└── APPENDIX_A_methods.md …        ← hand-authored appendices, committed
artifacts/
├── *.json                         ← engine results, committed (plain git)
├── *.parquet                      ← data tables, committed (plain git)
└── figures/*.png                  ← plots, committed (plain git)
```

## The stable-override pattern

**Important**: jellycell ≥ 1.4.0 (see `pyproject.toml`) ships a
`jellycell.tearsheets` Python API — `jt.findings()`, `jt.methodology()`,
`jt.audit()` (see `jellycell-gotchas.md` §5). That is the preferred entry
point for *new* notebooks. There is **no** `_helpers.reporting` module in
this repo — ignore any older reference to `emit_findings_markdown`.

This study's committed `manuscripts/FINDINGS.md` predates that API. It is
emitted by the `findings_md` cell of `notebooks/06_synthesis.py`, which
reads `artifacts/reconciled_findings.json`, builds the markdown inline
against a module-level `STABLE_OVERRIDES` dict, and writes the file. The
hand-rolled form is kept as-is for byte-stability of the already-committed
output:

```python
# notebooks/06_synthesis.py
STABLE_OVERRIDES = {
    "project": "case-study-rat-containerization",
    "generated_at": "stable",
    "hostname": "showcase-runner",
    "author": "Blaise Albis-Burdige",
    "author_url": "https://blaiseab.com",
    "month_year": "July 2026",   # bump per release, not per run
    "version": "4.0.0",
}

rec = json.loads(open("artifacts/reconciled_findings.json").read())
lines = [f"# Findings — {STABLE_OVERRIDES['project']}", ""]
# … build every row from `rec`; pin every nondeterministic field to STABLE_OVERRIDES …
Path("manuscripts/FINDINGS.md").write_text("\n".join(lines))
```

For a **new** notebook, prefer the 1.4.0 API over hand-rolling (matches
`jellycell-gotchas.md` §5):

```python
import jellycell.tearsheets as jt

jt.findings(
    results=rec["headline_att"],   # e.g. {"bjs": {"att": -11.93, "se": 0.65, ...}}
    out_path="manuscripts/FINDINGS.md",
    project="case-study-rat-containerization",
    template_overrides={"author": "Blaise Albis-Burdige", "month_year": "July 2026"},
)
```

**Rule**: every field that would be nondeterministic → pinned to a stable literal. The result is a `FINDINGS.md` whose git diff between runs is empty unless the underlying `results` numbers changed.

Per-notebook tearsheets (`manuscripts/tearsheets/*.md`) are exported via
the CLI:

```bash
uv run jellycell export tearsheet notebooks/03_main_effects.py \
    --output manuscripts/tearsheets/03_main_effects.md
```

`jellycell export tearsheet` is its own command — `jellycell render` builds
the HTML catalogue under `site/` and does **not** rewrite the committed
markdown. (Each tearsheet carries a `last run` timestamp line, so that one
line is expected to move when you re-export.)

## Canonical overrides dict

```python
STABLE_OVERRIDES = {
    "project": "case-study-rat-containerization",
    "generated_at": "stable",
    "hostname": "showcase-runner",  # historical value, intentional — baked into committed tearsheets; changing it dirties every regen
    "author": "Blaise Albis-Burdige",
    "author_url": "https://blaiseab.com",
    "month_year": "July 2026",   # bump per release, not per run
    "version": "4.0.0",
}
```

Keep one `STABLE_OVERRIDES` constant at the top of the findings cell in
`notebooks/06_synthesis.py`.

## Byte-identical regen check

Re-run the emitting notebook (not `jellycell render`, which only rebuilds
`site/`) and diff:

```bash
cp manuscripts/FINDINGS.md /tmp/before.md
uv run jellycell run notebooks/06_synthesis.py
diff -u /tmp/before.md manuscripts/FINDINGS.md
# empty diff expected
```

If the diff isn't empty, find the non-stable override and pin it. Common
culprits: `datetime.now()`, `socket.gethostname()`, `uuid.uuid4()`,
`random.seed(None)`, floating-point last-digit drift from `numpy`
(set `np.random.seed(42)` at the top of every notebook).

## Floating-point determinism

Stable overrides aren't enough — the *underlying numbers* also need to be
deterministic. Seed every random number generator at the top of every
notebook:

```python
import numpy as np
import random

np.random.seed(42)
random.seed(42)
```

For bootstrap / permutation tests:

```python
from factor_factory.engines.diagnostics import bootstrap_ci

ci = bootstrap_ci(panel, statistic=lambda p: p.mean(), n_boot=1000, seed=42)
```

For parallel execution (some engines use `joblib`):

```python
result = did_estimate(panel, ..., n_jobs=1)  # serialized = reproducible
```

## Committed binaries — separate concern

Figures, parquet, feather, and geojson are committed as **plain git blobs**
— this repo uses **no** Git LFS (see `.gitattributes`: the binary formats
carry `!text !filter !merge !diff` only to stop git mangling them, and the
header notes they are "left over from the LFS export"). Keep each committed
artifact under the `max_committed_size_mb = 50` cap in `jellycell.toml`.
Because the blob *is* the file, a figure that rendered deterministically
produces an identical blob (empty diff). Seed your plots:

```python
import matplotlib.pyplot as plt
plt.rcParams["svg.hashsalt"] = "showcase"   # stable SVG IDs (historical salt value, intentional — keep as-is)
# or for PNG: matplotlib is already deterministic if data + style are fixed
```

## Tearsheets ≠ the paper

- `FINDINGS.md`, `DIAGNOSTICS_CHECKLIST.md`, and `tearsheets/*.md` = auto-generated, committed.
- `MANUSCRIPT.md` = hand-authored paper prose (see `paper-cadence` skill), never templated.
- `AUDIT.md`, `METHODOLOGY.md`, and the appendices = hand-authored, never templated.

Do not hand-edit `FINDINGS.md` — edits get overwritten on next run. Put
prose in `MANUSCRIPT.md`; put table/number generation logic in
`notebooks/06_synthesis.py`, which emits the `artifacts/*.json` the
tearsheet reads.

## When to invoke this skill

- Setting up the `manuscripts/` dir.
- Seeing git noise in `FINDINGS.md` after re-running `notebooks/06_synthesis.py` with no real changes.
- Reviewing a PR that edits `FINDINGS.md` — reject hand edits, redirect to notebook changes.

## When NOT to invoke this skill

- `README.md` or `AUDIT.md` — both are hand-authored, no templating involved.
- `MANUSCRIPT.md` — hand-authored, follows `paper-cadence` skill instead.
