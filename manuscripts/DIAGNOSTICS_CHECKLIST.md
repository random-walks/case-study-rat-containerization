# Diagnostics checklist — rat-containerization

This checklist documents which assumptions of the staggered-DiD
identification strategy were tested, where the results live, and how
to interpret each pass/fail. The live numbers come from
`artifacts/diagnostic_checklist.parquet` (regenerated by
`06_synthesis_and_publication.py`).

| Diagnostic | Test | Notebook | Status interpretation |
|---|---|---|---|
| **Pre-treatment balance** | Standardized mean differences (SMD) per covariate, treated vs. control | `02_balance_and_pretrends.py` | `\|SMD\| < 0.5` acceptable; `< 0.1` ideal. Severe imbalance flags omitted-variable concerns. |
| **Parallel trends (visual)** | Group-mean line plot, treated + control through 2024 with pilot date marked | `02_balance_and_pretrends.py` (figure) | Lines should track in parallel pre-June 2024; visible divergence post-June is the treatment effect. |
| **Multi-estimator agreement** | TWFE + Callaway-Sant'Anna + synthetic control side-by-side | `03_main_effects.py` | Convergence across estimators is the strongest robustness check. Divergence triggers `mechanism` analysis (notebook 05). |
| **Residual normality** | Jarque-Bera + Shapiro-Wilk on TWFE residuals | `04_diagnostics.py` | `p > 0.05` supports normal residuals. Failure suggests non-Gaussian errors; clustered SE is robust to this but interpretation should note. |
| **Heteroskedasticity (visual)** | Residuals-vs-fitted + Q-Q | `04_diagnostics.py` (figure) | Q-Q deviation in tails or fan-shaped residuals → cluster-robust SE preferred over OLS SE (which we already use). |
| **Leave-one-treated-out jackknife** | Drop each treated CD; refit; report ATT range | `04_diagnostics.py` | Tight range (`max - min` small) means no single unit drives the ATT. Wide range → influential-unit problem. |
| **Block bootstrap CI** | Resample districts (B=200), refit TWFE, percentile CI | `04_diagnostics.py` | Bootstrap CI close to analytic CI = good agreement. Bootstrap CI much wider → analytic SE understates uncertainty. |
| **HTE by baseline volume** | Stratify by pre-treatment median; refit per subgroup | `05_robustness_and_mechanism.py` | Subgroup ATTs should be in same direction for "average effect" interpretation. Diverging subgroup signs → policy effect is not uniform. |
| **Placebo test (pre-treatment)** | Treatment shifted to Jan 2024, restricted to pre-true-treatment data | `05_robustness_and_mechanism.py` | `p > 0.05` on placebo ATT = parallel trends hold. `p < 0.05` = pre-treatment differential trend, parallel trends violated. |
| **Seasonal robustness** | Refit on district-demeaned outcome (within-unit deviation) | `05_robustness_and_mechanism.py` | ATT should be similar in magnitude/sign to raw TWFE; large divergence suggests seasonal cycle drives result. |

## Diagnostics NOT in this showcase

The following are useful diagnostics that the *next* iteration of this
analysis should add. They are out of scope for this showcase only
because of time / data constraints:

- **Spatial spillover test (SUTVA).** Spatial-lag DiD or treatment-on-
  treated decomposition. Requires real spatial weights at CD level —
  tractable but adds another notebook.
- **Proper local-polynomial RDD.** `rdrobust` package + IK
  bandwidth + polynomial sensitivity. Requires lat/lon for individual
  complaints (the panel here aggregates to CD level; per-record lat/lon
  is in the source CSVs but discarded by `build_complaint_panel`).
- **Multi-year control series.** A "true" parallel-trends test needs
  pre-treatment data (2022 + 2023 ideally) to plot multiple years of
  control trajectory. Adding `start_date='2022-01-01'` to the data fetch
  closes this gap — ~25 MB additional data, ~3 min extra fetch on first
  run.
- **Reporting-bias channel.** Upstream's `latent_reporting_bias_em`
  attempts to separate observed counts into (true rate × reporting
  probability). With 1 year of data the EM is underdetermined; revisit
  once multi-year data is in.
- **Multiple-comparison correction.** Notebook 05's HTE strata + future
  spatial subgroup tests should pool into one Benjamini-Hochberg
  adjustment.
- **Power analysis.** `nyc311.stats.minimum_detectable_effect` —
  important for interpreting null results. Was in upstream's notebook
  09; omitted here for time, easy to add.
